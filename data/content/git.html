Gitignore

hello.*
matches any file or folder whose name begins with hello (matches hello.txt, hello)
/hello.*
restrict to the directory of this file (matches hello.txt, hello.c but not a/hello.java`)
foo/
matches a directory foo and paths underneath it, but will not match a regular file or a symbolic link foo
doc/frotz and /doc/frotz
both have the same effect, a leading slash is not relevant if there is already a middle slash in the pattern
foo/*
matches foo/test.json (a regular file), foo/bar (a directory) does NOT match foo/bar/hello.c (a regular file). The asterisk in the pattern does not match bar/hello.c which has a slash in it
